# Diary

这是一个用来记录日常事项、技术折腾、阶段性总结与临时备忘的仓库。

## 目录结构

```text
daily/
  2026/
    2026-04-05-K20-Pro-Gemma4-部署记录.md
```

## 命名规则

日记文件统一使用下面的格式：

```text
YYYY-MM-DD-事件名称.md
```

例如：

```text
2026-04-05-K20-Pro-Gemma4-部署记录.md
```

这样做的好处是：

- 按文件名即可直接按时间排序
- 一眼就能知道当天记录的主题
- 后续按年份扩展也比较自然
